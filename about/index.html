🚀 成長與專案經驗

以下是我在學習與實作上深刻影響、讓我持續成長的里程碑。

📌 2022 年 iThome 鐵人賽：挑戰自我、持續精進

2022 年，我參加了 iThome 鐵人賽 — 一個需要連續 30 天每日產出的技術挑戰。
我選擇「三十天全端工程師挑戰」為主題，在 Raspberry Pi 上打造購物商城系統，包含前端、後端、資料庫、部署全部自己完成。

過程中幾乎是「不眠不休」，但也是我成長最快的時期之一，提升了專案管理能力，也真正體會到動手做的力量。

📌 Raspberry Pi 部署：從硬體到後端的首次整合

學習後端初期，我用 Raspberry Pi 建置了個人伺服器，撰寫爬蟲收集數據、自動化整理並寄出報表。部署過程中包含：

  • 安裝作業系統
  • 配置網路、Port Forward
  • 部署後端服務
  • 自動化排程與資料儲存

2024 年因為租屋爬蟲專案，被 Joeman 採訪，也是非常有趣的人生經驗。

📌 GitLab CI/CD：把部署流程自動化

為了讓自己的專案更穩定、更可靠，我參與 GitLab CI/CD 課程，學習：

  • Docker 自動建置 Image
  • Docker Compose 自動部署
  • GitLab Runner 架設
  • pipeline 自動化設計

並成功將這些技術導入工作項目，提升部署速度與可靠度。

📌 2025 年：協助單位自建 GitLab，打造更自主的開發流程

2025 年，我協助單位規劃並架設了內部自建 GitLab：

  • 建立 GitLab 伺服器
  • 設定專案群組、權限、Runner
  • 設計自動化部署流程
  • 協助同仁從傳統開發模式轉向 CI/CD

自建 GitLab 讓團隊不再依賴外部服務，部署速度、程式管理與更新流程更加一致、透明且高效。
也讓更多同仁開始接觸自動化，大幅改善了團隊開發體驗。

📌 監控與自動化實戰：從網路架構規劃到 13 台伺服器監控建置

在上完 CCNA 課程後，我對整體網路架構的概念有了顯著提升。為了強化服務的穩定性，我主導了監控系統的建置專案：

  • 規劃優先：在實際架設前，我特別繪製了詳細的架構圖。圖中清晰標註了所有 Server 的角色、Prometheus 的入口位置，以及每台 Server 需要開放給 Prometheus 抓取資料（target）的 Port 號。這份嚴謹的規劃讓主管為之驚豔，也確保了後續執行過程中的高準確性。
  • 網路安全控管：根據架構圖，我對 UFW 防火牆 進行了精確設定，清楚定義每個方向的 allow 與 deny 規則，確保監控流量安全且正確。
  • 工項拆解與持續推進：有了明確的架構圖作為藍圖，我得以將龐大的監控架構拆解成多個小工項。這讓我能在日常繁忙的開發工作之餘，每天都能有持續的進展。
  • Ansible 自動化加速：透過 Ansible 進行伺服器的套件管理與設定，我僅用了約一個月的時間，就成功完成了 13 台伺服器的監控部署（Prometheus + Grafana + Node Exporter）。

這段經驗不僅提升了我對系統維運的掌握度，更證明了「先規劃、後實作」與「自動化工具」在大型架構管理中的核心價值。

📌 AI 應用與 RAG 建置：Side Project 的重要里程碑

AI 是近年我最投入的技術領域之一。我租用雲端主機，架設後端服務，完整打造：

  • GPT API 串接
  • 資料分段（chunk）
  • 向量化
  • 向量資料庫建立
  • 查詢與生成流程
  • LINE Bot 自動回覆整合

透過這個專案，我更深入掌握大型語言模型與應用系統整合的實務，也奠定我往 AI 工程領域前進的重要基礎。

📌 前端學習：拓展技能、更好理解整體架構
