後端工程師的網通入坑之路：為什麼我推薦每個後端都該懂CCNA

職涯規劃 CCNA Networking
 2025/12/08  次瀏覽  Share
  • 
"""

最近，我終於完成了一個放在心上許久的目標——去上了CCNA認證課程。當我跟身邊的工程師朋友、同事提到這個決定時，不少人露出了疑惑的表情：「一個後端工程師，為什麼要去學那個？我們又不是要當網管。」

這篇文章，我想分享我的心路歷程，以及為什麼我現在會斬釘截鐵地說：「如果你是個有企圖心的後端工程師，你真的該去了解CCNA在談些什麼。」

旅程的起點：一個 Docker 網路衝突

這一切的開端，源自於工作上一個看似單純卻又棘手的問題。當時，我正在本地開發環境中運行一個 Docker 容器，卻發現它內部的網段(CIDR)正好與公司內網的某個重要服務發生了衝突，導致我無法從容器中存取該服務。

為了解決這個問題，我花了很多時間去查資料、問同事，雖然最終靠著修改 Docker 的網路配置繞過了，但我心中那個最大的疑問始終揮之不去：「這一切底下，究竟是怎麼運作的？為什麼會衝突？網段又是什麼？」我知道，我對網路的理解，還停留在非常表層的階段。

被質疑的決定與現實的殘酷

抱著解決心中疑惑的渴望，我決定報名恆逸的CCNA課程。這是一個不小的決定，不僅僅是因為那高達六萬多的學費，更是因為身邊的質疑聲浪。

朋友和同事不解地問：「你是後-端-工-程-師耶，學這個做什麼？把時間拿去刷 LeetCode、研究新的框架不是更好嗎？」

他們的質疑並非全無道理。如果你是一個只負責純粹演算法、數據結構，完全不用接觸伺服器、不用管部署流程的後端工程師，那麼專注在時間複雜度、空間複雜度，並熟練使用各種開發套件，或許真的足夠了。

但現實是，有多少工作能切分得如此乾淨？身為後端，我們或多或少都會碰到：

  • 伺服器部署與維運
  • Docker 與容器化技術
  • CI/CD 流程的建立
  • 雲端服務 (AWS, GCP, Azure) 的網路設定
  • API Gateway 的路由規則
  • 資料庫的連線問題

這些環節，每一個都與「網路」緊密相連。當問題發生時，只會說「在我電腦上是好的」，是遠遠不夠的。

豁然開朗：串起知識的珍珠鍊

整個CCNA課程對我來說，與其說是學習一門全新的知識，更像是一個「串連」的過程。過去那些在工作中零星碰到、一知半解的名詞，如 IP、Subnet Mask、Gateway、DNS、Port… 終於在這堂課中被有系統地串成了一條完整的珍珠鍊。

我終於搞懂了：

  • **網路七層 (OSI Model)**：讓我知道問題排查時，該從哪個層次開始思考。
  • IP位址與子網路劃分：徹底理解了為何會有網段衝突，以及如何進行規劃。
  • **路由協定 (Routing Protocol)**：明白了數據包(Packet)是如何在茫茫網海中找到最佳路徑。
  • **交換機 (Switch) 與路由器 (Router)**：懂得了它們在網路中的各自職責與運作原理。
  • **ACL (Access Control List)**：學習到如何制定規則來控制流量，保障網路安全。
  • Traceroute 的奧秘：以前只會用這個指令，現在終於知道它背後是利用ICMP與TTL來追蹤路徑。

而最讓我驚豔的，是這門課也為我打開了資安的大門。

不只是網通，更是資安的基石

在公司內部的資安訓練中，講師總是告誡我們：「不要隨便連別人的Wi-Fi，也不要把自己的Wi-Fi給別人用」，並提到了「中間人攻擊 (Man-in-the-Middle)」。當時我只知道要照做，卻不知道「為什麼」。

上了課才知道，光是一個基礎到不行的「ARP (位址解析協定)」，就存在著能被輕易利用的弱點。攻擊者可以在同一個區域網路內，透過 ARP 欺騙，讓自己偽裝成Gateway，從而攔截、竊聽你所有的網路流量。這讓我瞬間理解了公共Wi-Fi的危險性，背後發涼。

給後端夥伴們的建議

常常有人覺得，學這些理論，但未來又不會真的去設定 Cisco 的路由器或交換機。但我想說的是，網路運作的原理是通用的。你在 CCNA 學到的概念，在 Linux 系統中也完全適用，只是實現的工具和指令不同而已。例如：

  • 在 Cisco IOS 你用 show ip route，在 Linux 你用 ip route 或 route -n。
  • Cisco 的 ACL 概念，與 Linux 的 iptables 或 firewalld 有異曲同工之妙。

我承認，因為我本身熟悉 Linux 操作，所以在課程中對很多指令和概念能更快上手。對於沒有 Linux 基礎的同學，可能會感到稍微吃力一些。但這也反過來證明了，身為一個現代後端工程師，Linux 和網路知識是相輔相成的。

結論

所以，後端工程師到底需不需要學CCNA？

我的答案是：絕對推薦。

這筆投資換來的，不是讓你轉行當網管，而是讓你成為一個更全面、更具備問題解決能力的工程師。當下一次服務連不上、網路延遲變高、或是需要規劃一個新的系統架構時，你的腦中會有更清晰的藍圖，而不是只能模糊地猜測。

這是一條能讓你從「只會寫Code的開發者」蛻變為「能駕馭系統的工程師」的必經之路。

"""
